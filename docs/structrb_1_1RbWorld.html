Detailed Description

Owner of all rigid body dynamics values.

Struct provides access to values and ensures lifetimes

Why World?
World exists to house the DynamicBodies and robots together. Robots store indexes into DynamicsBodies, so this way we can ensure there are no null references,and we can manage the lifecycle of the dynamics bodies and robots together
Construction
To construct RbWorld, initialize use default construction and then add robots and landmarks as you see fit. The only requirement is createRngEngine must be called before advancing the world to ensure the rng engine is initialized.
