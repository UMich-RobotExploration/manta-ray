69/**
70 * @brief Computes the relative transform of pose with respect to refPose
71 * Poses must be relative to the same frame otherwise this is meaningless.
72 *
73 * @par Math:
74 * Reference: Modern Robotics, Section 3.3.1.2, "Changing the reference frame"
75 * Normally we are given the equation as follows:
76 * \f[
77 * T_{ac} = T_{ab} \cdot T_{bc}
78 * \f]
79 * Where b is canceled, and we get T_ac as the result. However, we want the
80 * relative transform. If we treat pose as T_ac and refPose as T_bc, we can see
81 * we want to solve for T_ab. We can do this by:
82 * \f[
83 * T_{ac} \cdot T_{bc}^{-1} = T_{ab}
84 * \f]
85 *
86 * @param pose Pose we want to express relative to refPose
87 * @param refPose "Fixed" pose we are computing relative to
88 * @param outputPose Output pose expressed in refPose frame
89 */
90void relativeTransform(const manif::SE3d &pose, const manif::SE3d &refPose,
91 manif::SE3d &outputPose);
