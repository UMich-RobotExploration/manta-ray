Covariance Serialization

PFG covariance format: column-major lower-triangular unique elements.

A symmetric NxN covariance matrix is serialized as N*(N+1)/2 space-separated values. The ordering is column-major lower-triangular: iterate over columns left-to-right, and within each column emit rows from the diagonal downward.

This matches py_factor_graph.utils.matrix_utils:

  • get_list_column_major_from_symmetric_matrix() (writer)
  • get_symmetric_matrix_from_list_column_major() (reader)

For a 3x3 matrix with indices (row, col):

c0 = (0,0)
c1 = (1,0) c3 = (1,1)
c2 = (2,0) c4 = (2,1) c5 = (2,2)

Emitted order: c0 c1 c2 c3 c4 c5 (6 elements)

For a 6x6 matrix (pose covariance, tangent-space ordering [x, y, z, roll, pitch, yaw]):

c0 = (0,0)
c1 = (1,0) c6 = (1,1)
c2 = (2,0) c7 = (2,1) c11 = (2,2)
c3 = (3,0) c8 = (3,1) c12 = (3,2) c15 = (3,3)
c4 = (4,0) c9 = (4,1) c13 = (4,2) c16 = (4,3) c18 = (4,4)
c5 = (5,0) c10 = (5,1) c14 = (5,2) c17 = (5,3) c19 = (5,4) c20 =
(5,5)

Emitted order: c0 c1 c2 c3 c4 c5 c6 c7 ... c20 (21 elements)

Diagonal indices for a diagonal-only 6x6: {0, 6, 11, 15, 18, 20} Diagonal indices for a diagonal-only 3x3: {0, 3, 5}

Note
For diagonal matrices the column-major lower-triangular and row-major upper-triangular orderings produce identical output because all off-diagonal elements are zero.

Writes a PFG factor graph file from simulation data.

Iterates over all robots and their sensors to produce a complete factor graph in PFG text format. Sections are written in the order required by the PFG spec: pose vertices, landmark vertices, pose priors, landmark priors, odometry edges, then range edges.

Parameters
filenameOutput file path (e.g., "output.pfg")
worldThe simulation world (robots with sensors, landmarks)
measurementsRange measurements from AcousticPairwiseRangeSystem
configWriter configuration (covariance params, GT vs odom flag)
